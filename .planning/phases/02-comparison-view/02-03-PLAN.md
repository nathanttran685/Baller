    **1. Create `src/app/compare/utils/diffUtils.ts`:**

    Implement the pure diff computation functions following the research code example exactly:

    ```typescript
    export interface PriceDiff {
      leftPrice: number | null;
      rightPrice: number | null;
      difference: number | null;
      cheaperSide: 'left' | 'right' | 'equal' | null;
    }

    export interface ConditionDiff {
      leftScore: number | null;
      rightScore: number | null;
      leftLabel: string | null;
      rightLabel: string | null;
      betterSide: 'left' | 'right' | 'equal' | null;
    }
    ```

    `computePriceDiff(leftPrice: string | undefined, rightPrice: string | undefined): PriceDiff`
    - Parse prices using same logic as DashboardClient's `parsePriceToNumber`: strip non-digit/dot chars, parse as number, return null if <= 0 or NaN.
    - If either is null, return `{ ..., difference: null, cheaperSide: null }`.
    - Compute `difference = Math.abs(left - right)`, `cheaperSide` based on which is lower.

    `computeConditionDiff(leftScore: number | undefined, leftLabel: string | undefined, rightScore: number | undefined, rightLabel: string | undefined): ConditionDiff`
    - If either score is undefined/null, return `betterSide: null`.
    - Compare scores numerically. Higher = better.

    **2. Create `src/app/compare/components/PriceComparison.tsx`:**

    Props: `{ leftPrice: string | undefined; rightPrice: string | undefined; }`

    Uses `computePriceDiff` internally. Renders a row with:
    - Left price with arrow indicator: `ArrowDown` (green) if cheaper, `ArrowUp` (red) if more expensive, no arrow if equal
    - Dollar difference in center: "$150 difference" in a bold badge
    - Right price with opposite arrow
    - If no price data available, render nothing

    Arrow styling: Arrow icons from lucide-react (`ArrowUp`, `ArrowDown`), size-5. Green `text-[#00FF00]` for cheaper, red `text-[#FF0000]` for more expensive. Background behind each price: white card with `${b5} ${shadow4} ${roundedXl} px-4 py-2`.

    Layout: `flex items-center justify-between gap-4` within a full-width row.

    **3. Create `src/app/compare/components/ConditionComparison.tsx`:**

    Props: `{ leftScore?: number; leftLabel?: string; rightScore?: number; rightLabel?: string; }`

    Uses `computeConditionDiff` internally. Renders side-by-side condition bars:
    - Each side: a horizontal progress bar (same pattern as research's ConditionBar example)
      - Bar container: `h-4 w-full ${b5} ${roundedXl} bg-gray-200 overflow-hidden`
      - Fill: colored based on score (>= 0.8 `bg-[#00FF00]`, >= 0.6 `bg-[#FADF0B]`, >= 0.4 `bg-[#FF6600]`, < 0.4 `bg-[#FF0000]`)
      - Width: `{Math.round(score * 100)}%`
    - Label above each bar: condition label + percentage in `${space} text-sm font-bold uppercase`
    - If either score is missing, show "N/A" text instead of bar

    Layout: `grid grid-cols-2 gap-6` for side-by-side bars.
